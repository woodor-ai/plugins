# 0.8.25 全局超管身份契约（locked）

定稿：plugins-wic，2026-06-20。对接方：amb-wic（amp + PWA）。

## 设计总纲
身份模型仍是 `(project, name)` 复合主键。新增一类**全局身份**：`project = "*"`（单星号，非 NULL、非空串）。复合键原样吃下它，messages / group_members / read_cursors 的 plumbing 一行不改，只在注册 / resolve / 显示三处特判，外加推导端一道防撞清洗。

## ① 注册（registration）
- 新增 CLI 标志：`meeting online <name> --global`。设了就**跳过 cwd→project 推导**，直接 `project="*"`；cwd 照常记录（仅参考），不参与 project。
- daemon `_register` 原样接受 `project="*"`（PK `(*,name)` 即唯一性），无需额外逻辑。
- `monitor.py` 透传 `--global`（与现有 `--director` 同机制），常驻 monitor 重注册时带上。
- amp 给 Tommy 的 self-session 改成 `online Tommy --global`（替换现 `Tommy@AMBridge`）。

## ② resolve / 寻址
- 裸名 `<X>`：若存在全局身份 `(*, X)`，**无歧义解析到它**（全局身份优先于任何同名 project 行，不抛 ambiguity）。
- 显式 `<X>@*` 直接寻址全局身份。
- 全局名由 PK `(*, X)` 保证全局唯一；不额外禁止 `X@proj` 共存，但 resolve 一律偏向全局。

## ③ 显示
- `project == "*"` 一律渲染成**裸 `<name>`**（去掉 `@` 后缀）：daemon 文本（show/turn）、CLI list/read、客户端（PWA/amp 把 `@*` 藏掉）。
- JSON 字段照常带 `sender_project="*"` 字面值；客户端特判 `"*"` → 隐藏。

## ④ 防撞清洗（关键，补 explore 发现的坑）
- `_derive_project`（meeting:106-119 + monitor.py:67-80）：算完 basename 后，**若结果 == `"*"` 则强制改写为 `"_"`**。推导端从此永不可能吐出 sentinel；`"*"` 只可能由 `--global` 显式产生。两道独立护栏。

## ⑤ 迁移（0.8.25 cutover，非 wipe）
- 一次性 SQL：把 Tommy 现有行 `project AMBridge→*`，范围限 `sessions` + `group_members`（让他现有群成员关系平移到全局身份）。
- `messages` 历史**不改**（append-only；旧消息保留 sender_project=AMBridge）。此后 Tommy 以 `Tommy@*` 发。
- amp 同步把注册改 `--global`。两步一起切。

## 边界
- 目前只有 Tommy 是全局；机制通用（任意 `--global` 名）。
- **群本身不全局**——群仍属某 project；全局身份是跨项目群的成员。
- read_cursors / messages plumbing 不动。
- sentinel 取值 `"*"`：basename 合法字符集里虽可能出现，但经 ④ 清洗后推导端不可达，唯一来源是 `--global`。
